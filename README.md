## έεο &nbsp; Platform-Wide Systems

<br/>

### ΏθΦΡ &nbsp; Authentication & Role-Based Access Control

JWT-based authentication with email + password. Role stored in the token payload έΑΦ validated on every protected API route and frontend portal guard. Seven distinct roles form a strict access hierarchy:

```
SUPER_ADMIN  έΗΡ  Full system access
    έΗΥ
ADMIN        έΗΡ  Operations, inventory, staff, reservations
    έΗΥ
CHEF / KITCHEN_STAFF  έΗΡ  Kitchen queue only
CASHIER               έΗΡ  POS terminal only
DELIVERY_STAFF        έΗΡ  Delivery assignments only
    έΗΥ
CUSTOMER     έΗΡ  Public-facing portal
```

<br/>

### ΏθΦΦ &nbsp; Real-Time Notification Engine

- Notifications stored per-user on the backend
- Frontend polls `GET /api/auth/notifications` every 3 seconds
- Bell icon in navbar shows live unread count badge
- Auto-cleared from backend after delivery έΑΦ one-time read model
- Triggers: order status changes, reservation confirmations, low-stock alerts (admin only), system messages

<br/>

### ΏθΩΕ &nbsp; Dual-Mode Persistence

```
Production Mode  έΗΤ  PostgreSQL via Prisma ORM
                     έΦΦέΦΑ Full schema, migrations, relational integrity

Offline Mode     έΗΤ  JSON Mock Database (auto-generated)
                     έΦΦέΦΑ mock-db-persistence.json
                     έΦΦέΦΑ Saved every 2 seconds via setInterval
                     έΦΦέΦΑ Full feature parity έΑΦ zero degradation
                     έΦΦέΦΑ Delete file to reset all data
```
